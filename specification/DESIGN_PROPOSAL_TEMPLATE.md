# 详细设计方案模板（社区评审 + AI 协作版）

## 使用说明

本模板用于社区协作开发流程：`需求提出 -> 方案设计 -> 社区评审 -> 开发调试 -> 合并发布`。

适用对象：

- 新加入社区的开发者
- 需要提交设计方案再开发的贡献者
- 需要快速审阅和追踪设计决策的核心维护者

建议规则：

- AI 可参与起草，但结论必须可追溯到项目事实（代码、历史决策、性能数据、社区规范、讨论记录）
- 方案文档必须区分「AI 输出」和「人工最终判断」
- 没有回滚方案和验证计划的设计，不进入开发阶段

## 附录 B：提示词模板（新人可直接复用）

### B.1 需求澄清

```text
你是该项目的资深维护者。请基于以下需求，输出：
1. 问题定义（可验证）
2. 功能/非功能需求拆解
3. 明确非目标
4. 需要向提需求方追问的问题（最多5个）
需求如下：<<<粘贴需求>>>
```

### B.2 候选方案对比

```text
请给出至少2种可实施方案，并按以下维度表格对比：
复杂度、侵入性、性能、风险、回滚难度、长期维护成本。
最后给出“推荐方案+不推荐其他方案的理由”。
上下文：<<<现有架构/约束>>>
```

### B.3 详细设计展开

```text
请基于“推荐方案”输出详细设计：
- 模块职责变化
- 接口定义（请求/响应/错误码）
- 数据模型变更
- 失败场景与重试/降级策略
- 发布与回滚步骤
要求：尽量贴合已有技术栈，不新增不必要基础设施。
```

### B.4 测试计划生成

```text
请从高风险路径倒推测试用例，输出：
- 单测清单
- 集成测试清单
- 回归重点
- 验收用例（Given-When-Then）
并标注每条用例覆盖的风险点。
```

### B.5 评审前自检

```text
请扮演严格评审人，指出该方案的：
- 模糊描述
- 缺失证据
- 不可执行步骤
- 回滚风险
并给出最小修复建议。
```
