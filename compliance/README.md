
# LDAP

Lightweight Directory Access Protocol，轻量级目录访问协议，是一种用于访问和维护分布式目录信息服务的应用层协议。

## 核心特点

- 目录服务：类似电话簿，存储组织化的信息
- 树形结构：数据以层次化方式组织
- 读多写少：优化了查询性能
- 标准协议：基于 TCP/IP，通常使用 389 端口（LDAPS 使用 636）

LDAP：存储和管理数据的"仓库"和"协议"
RBAC/ABAC：使用数据进行权限判断的"规则"和"模型"

# RBAC

Role-Based Access Control，基于角色的访问控制，不直接给用户分配权限，而是通过角色间接授权

## 基本模型

用户(User) → 角色(Role) → 权限(Permission) → 资源(Resource)

┌─────────┐         ┌─────────┐         ┌─────────┐
│  张三   │────────>│  经理   │────────>│ 审批权限│
│  李四   │         │  角色   │         │ 查看权限│
└─────────┘         └─────────┘         └─────────┘
   用户              角色                权限

一个用户可以有多个角色
一个角色可以有多个权限
一个权限可以分配给多个角色

## 应用场景

企业内部、电商平台


# ABAC

Attribute-Based Access Control，基于属性的动态策略评估，根据主体、客体、环境的属性，动态评估是否授权。

## 四类属性

┌──────────────────────────────────────────┐
│  主体属性 (Subject Attributes)           │
│  - 用户部门、职级、地理位置等            │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│  客体属性 (Object Attributes)            │
│  - 资源类型、敏感级别、所有者等          │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│  环境属性 (Environment Attributes)       │
│  - 时间、IP地址、设备类型等              │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│  操作属性 (Action Attributes)            │
│  - 读、写、删除、执行等                  │
└──────────────────────────────────────────┘

## 应用场景

医疗系统，金融系统，云服务平台
